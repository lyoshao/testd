# Дипломный проект по автоматизации тестирования

Автоматизация сквозного (E2E) UI-тестирования маркетплейса Wildberries, а также интеграционного тестирования REST API (на основе публичного тестового сервиса Fake Store API) с использованием архитектурного паттерна Page Object Model (POM).

## Технологический стек

* **Язык программирования:** Python 3.10+
* **Тестовый фреймворк:** Pytest
* **Тестирование UI:** Playwright (Python)
* **Тестирование API:** Requests
* **Отчетность:** Allure Framework

## Описание тестового покрытия

### 1. UI тесты (50 сценариев)

* **Поиск (15 тестов):** Проверка поиска товаров, автодополнения, фильтров, пагинации.
* **Каталог (15 тестов):** Проверка открытия каталога, категорий, подкатегорий.
* **Карточка товара (20 тестов):** Проверка названия, цены, корзины, избранного, характеристик.

### 2. API тесты (5 сценариев)

* **Получение списка продуктов:** Проверка GET /products
* **Получение продукта по ID:** Проверка GET /products/{id}
* **Получение категорий:** Проверка GET /products/categories
* **Успешная авторизация:** Проверка POST /auth/login
* **Неверный пароль:** Проверка POST /auth/login (401)

* **Вставка и выборка пользователя:** Проверка CRUD операций с SQLite

## Инструкция по запуску тестов локально
1. Клонирование репозитория:
    git clone https://github.com/lyoshao/Diploma-QAP
    cd Diploma-QAP
2. Создание и активация виртуального окружения:
    python -m venv venv
    venv\Scripts\activate
3. Установка зависимостей:
    pip install -r requirements.txt
4. Установка браузеров Playwright:
    playwright install
5. Запуск тестов:
    pytest

## Инструкция по генерации отчётов
Для генерации Allure отчета выполните:
    pytest --alluredir=allure-results
    allure serve allure-results

## Инструкция по запуску в Docker
1. Сборка Docker-образа:
    docker build -t Diploma-QAP
2. Запуск всех тестов:
    docker run --rm Diploma-QAP
